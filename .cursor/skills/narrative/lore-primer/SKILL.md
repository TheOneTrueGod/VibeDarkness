# Lore primer

1. **Read** `LORE.md` in this folder for the current high-level framing and any `[Canon]` / `[Draft]` / `[Open]` sections.
2. Respect **stability suffixes** on section headings (see `narrative-hub` skill): do not treat `[Open]` as fact or overwrite `[Canon]` without explicit user direction.
3. For **voice and format** by domain (enemies, abilities, characters, worlds), use the corresponding `writing-style-*` skill’s `STYLE.md` after checking this primer for conflicts.

If `LORE.md` is silent on a topic, treat that topic as **unwritten** until the user adds it—do not invent large Canon bridges without labeling them `[Draft]` and confirming.
